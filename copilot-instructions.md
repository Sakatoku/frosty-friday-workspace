# Copilot Instructions for Frosty Friday Workspace

## このドキュメントについて

- GitHub Copilotを始めとしたAIツール/AIエージェントが本リポジトリのコンテキストを理解しやすくするためのガイドです。
- 新しいコードや新しい文書を作成するとき、技術選定・設計方針・ディレクトリ構成を前提にしてください。
- 不確かな点がある場合は、本リポジトリ内のファイルを探索して、ユーザーに「こういうことですか？」と確認をするようにしてください。

## 前提条件

- 回答は必ず日本語でしてください。
- 本リポジトリはデータベースとしてSnowflakeを想定しています。
- コードについては、Snowflakeで動作するように用いるデータベース機能やSQL方言を注意深く選択してください。
- 文書については、最新のドキュメントを元に記載が正しいかを確認して、リファレンスを記述するようにしてください。
- 単一のファイルに対する編集のみが許可されています。複数のファイルに対する編集が必要な場合、まず何をするのか計画を立てた上で、ユーザーに「このような計画で進めようと思います。」と提案してください。このとき、ユーザーから計画の修正を求められた場合は計画を修正して、再提案をしてください。

## 重要な参照文献

- [Snowflakeの最新のドキュメント](https://docs.snowflake.com/)
- [Frosty Friday](https://frostyfriday.org/)

## ディレクトリ構成

本リポジトリは目的別に異なるディレクトリを採用しています。  

```
(root)
├── answers/                    # Frosty Fridayの解答を置くルートディレクトリ
│   └── FROSTYFRIDAY_WEEK{d}/   # 各チャレンジに対する解答
└── articles/                   # 各チャレンジに対する解答の解説記事

## ファイル名規則

本リポジトリでは主要なファイルに対して以下のファイル名規則を採用します。  

- 各チャレンジに対する解答。SQLファイルの場合：week{d}.sql
- 各チャレンジに対する解答。Notebookファイルの場合：FROSTYFRIDAY_WEEK{d}.ipynb
- 各チャレンジに対する解答の解説記事：week{d}.md
